STL, C++11

Szablony Funkcji

Załóżmy że chcemy sobie wyciągnąć maks z dwóch liczb - raz z intów, raz z double. Musimy napisać dwie fuunkcje.

int max(int a, int b) {
	return a > b ? a : b;
}

double max(double a, double b) {
	return a > b ? a : b;
}

Szablony pozwalają nam na napisanie tego samego przy pomocy

template <class T>
T max(T a, T b) {
	return a > b ? a : b;
}

i tadam!

#include <iostream>

using namespace std;

int main() {
	cout << max(5, 4) << endl;
	cout << max(3.14, 2.71) << endl;
	system("pause");
	return 0;
}

Szablony Klas

Podobnie możemy zadziałać na klasach:

template <class T>
class xd {
	T a, b;
public:
	xd(T first, T second) {
		a = first;
		b = second;
	}
	T maks();
};

template <class T>
T xd<T>::maks() {
	return a > b ? a : b;
}

Stworzona klasa xd ma dwa publiczne pola a i b. Konstruktor który nadaje im podane przez użytkownika wartości i funkcję maks z poprzedniego przykładu.

Cała zabawa polega jednak na tym że gdy tworzymy obiekt danej klasy, musimy nadać T jakąś wartość. I tak dla int wygląda to tak:

xd <int> obj(20, 10);
cout << obj.maks() << endl;

Dodajemy po typie obiektu jaki tworzymy podstawienie za T.

Specjalizacja szablonu klasy

A co w sytuacji gdy chcemy np. obsłużyć wszystkie liczby a nie np. litery.

Zaczynamy od szablony klasy:

template <class T>
class xd {
public:
	xd(T first) {
		cout << first << " is not a char!" << endl;
	}
};

A następnie obsługę charów:

template<>
class xd<char> {
public:
	xd(char first) {
		cout << first << " is indeed a char!" << endl;
	}
};

Jedyna zmiana polega na tym że tutaj juz konkretnie używamy char, a nie generyczne T.

Inicjalizacja List

Zamiast:

vector<int> v;
v.push_back(3);
v.push_back(4);
v.push_back(1);
v.push_back(9);

Można napisać:

vector<int> v = { 3, 4, 1, 9 };

Wszystkie kontenery STL obsługują tą funkcjonalność!

Incicjalizacja klasy

Zamiast:

class dog{
	public:
		int a;
		char b;
	};
dog d1 = { 5, 'q' };

Można:

class dog{
	public:
		dog(int a, char b) {}
	};
	dog d1 = { 5, 'q' };

Foreach

Dla vektora:

vector<int> v = { 3, 4, 1, 9 };

Możnabyło kiedyś wypisać zawartość za pomocą:

for (vector<int>::iterator itr = v.begin(); itr != v.end(); ++itr) {
	cout << (*itr);
}

Teraz można dla każdego obiektu z zdefiniowanym begin() i end() skorzystać z:

for (int i : v) {
	cout << i;
}

lub:

for (auto& i : v) {
	cout << i;
}

Dodanie nullptr

W przypadku dwóch funkcji:

void foo(int i) {
	/}/
}
void foo(int* i) {
	/}/
}

gdy wywoływaliśmy foo(NULL) kompilator nie wiedział co wywołać. Funkcję foo() z wskaźnikiem czy bez. Rozwiązaniem jest nullptr!

int main(){
	foo(nullptr)
}

Powtórne wykorzystanie konstruktora

Kiedyś gdy chcieliśmy mieć konstruktor domyślny i np. drugi dla int nie mogliśmy wywołać w drugim konstruktorze pierwszego i potem coś nowego dodać. Zamist tego istaniała praktyka deklarowania funkcji init() i potem korzystanie z niej w obu wypadkach:

class dog{
	init() {...};
public:
	dog(){
		init();
	}
	dog(int a){
		init();
		doOtherStuff();
	}
};

Teraz można zrobić to tak:

class dog{
public:
	dog(){
		stuff();
	}
	dog(int a) : dog(){
		doOtherStuff();
	}
};

Klasa final

Dodano słówko final do klasy, które sprawia że z danej klasy nie można dziedziczyć.

class A final {
public:
	int a = 5;
};

Konstruktor domyślny

Gdy mamy klasę i zdefinowany w niej konstruktor dla np. int, to kompilator nie tworzy już dodatkowo konstruktora domyślnego.

Przykład:

class dog {
	dog(int a) {}
};

Dodano teraz słówko default, które wymusza wygenerowanie domyślnego konstruktora:

class dog {
	dog(int a) {}
	dog() = default;
};

Constexpr

Wcześniej jak mieliśmy funkcję:

int foo() {
	return 3;
}

i chcemy zadeklarować 5-ellementową tablicę:

int tab[foo() + 3];

To zwracało nam błąd.

Teraz dzięki dodaniu słówku constexpr przed deklaracją typu funkcji całość skompiluje się bez problemu!

constexpr int foo() {
	return 3;
}

Nowe możliwości kodowania stringów

char* a = "string";			// normal string

char* b = u8"string";		// to define an UTF-8 string
char16_t* c = u"string";	// to define an UTF-16 string
char32_t* d = U"string";	// to define an UTF-32 string
char* e = R"(string)";		// to define raw string

Funkcja lambda

Anonimowa fuunkcja z możliwością wywołania:

auto f = [](int a, int b) {
	return a + b;
};

cout << f(3,4) << endl;

Polimorfizm

Pozwala nam na wyabstrahowanie pewnych zachowań od konkretnych typów danych. Dzięki niemu możemy wybrać zachowanie w zależności od kontekstu. Mówiąc ściślej, do jednej referencji można przypisać obiekty różnych typów, które dziedziczą po typie referencji, i wywołując przeciążoną metodę użyta zostanie metoda przypisanego typu referencji.

Dla odmiany rozpatrzmy przykład, w którym klasą bazową będzie Zwierze, a klasami pochodnymi będą Pies oraz Ryba. Klasa bazowa ma zadeklarowaną metodę (funkcję, którą możemy wywołać na rzecz obiektu) o nazwie dajGlos. Klasy pochodne mogą tę funkcję zdefiniować po swojemu i w ten sposób po wywołaniu metody dajGlos na obiekcie klasy Pies usłyszymy szczekanie, a po wywołaniu metody na obiekcie klasy Ryba usłyszymy tylko ciche bulgotanie, któremu towarzyszyć będzie bezgłośne osądzanie naszych wyborów życiowych.

class Zwierze {
protected:
	char* xd;
public:
	void setStr(char* a) {
		xd = a;
	}
};

class Pies: public Zwierze {
public:
	void dajGlos() {
		cout << xd << endl;
	}
};

class Ryba : public Zwierze {
public:
	void dajGlos() {
		cout << xd << endl;
	}
};

int main() {
	
	Pies p;
	Ryba r;

	Zwierze *z1 = &p;
	Zwierze *z2 = &r;

	z1->setStr("Hau, hau!");
	z2->setStr("Bul, bul, bul!");

	p.dajGlos();
	r.dajGlos();

	system("pause");
	return 0;
}

Funkcje wirtualne

Możemy w klasie Zwierze stworzyć wirtualną funkcję dajGlos:

class Zwierze {
protected:
	char* xd;
public:
	void setStr(char* a) {
		xd = a;
	}
	virtual void dajGlos() {}
};

Następnie klasy Pies i Ryba zostają bez zmian. Pozwala nam to na wywołanie funkcji dajGlos w taki sposób:

z1->dajGlos();
z2->dajGlos();

Klasy Abstrakcyjne

Klasy abstrakcyjne to, najogólniej, takie klasy dla których nie można stworzyć obiektu. Można by zadać sobie pytanie: "Po co więc nam taka klasa?". Odpowiedź jest prosta: aby z niej dziedziczyć.

class Zwierze {
public:
	virtual void dajGlos() {}
};
